## 2. 採用アイデア（1行で言い切る）
- これは何のアプリか: **「一枚の写真をAIが観察し、写った人や物から"住人・持ち主・状況"を名探偵のように推理する」Webアプリ「シャーロック・アイ」。**
- 選んだ理由（4基準のどこで勝つか）:
  - テーマ適合: 名探偵の本質＝観察推理そのものを体験にした。被写体を問わず成立。
  - 実装度（3時間で動くか）: ◎。入力＝写真1枚 / AI呼び出し＝1回 / 出力＝推理カード。状態管理が軽い。
  - チャレンジ精神（独創性）: 写真から人物像をその場で逆算する"観察芸"＋**答え合わせ（的中率・称号）**でゲーム性を付与。
  - AI活用度（プロダクト内でAIが何をするか）: Claude Visionが**その場で写真を読み推理を生成**。AIはプロセスでなくプロダクトの中核。

## 3. コア機能とAIの役割 ★最重要
- ユーザーがやること（入力）: 写真を1枚アップロード（部屋・デスク・カバンの中身・持ち物・人物・街角など何でも）→「観察を始める」を押す。
- AIがやること（プロダクト内のどこでAPIを呼ぶか）: フロント→`/api/claude`→Anthropic（Vision）。写真をbase64で送り、**観察推理型**（「〜のはずだ。なぜなら〜」）で人物像/状況を推理。写っていないものは推測しない縛り。
- 出力・見せ方:
  - `verdict`: 写真から読み取った人物像・状況を一言で（明朝大見出し）
  - `remark`: 名探偵の一言（古風で知的な口調）
  - `clues[]`（4つ）: 「観察（手がかり）→ 推理 → 確信度（高/中/低）」の**推理カード**を一枚ずつ捺印表示
  - **答え合わせ**: 各カードを「当たり／外れ」で判定 → 的中数を集計 → **的中率メダル＋称号**（4=真打・名探偵 … 0-1=迷探偵）
- ※「AIはプロセスではなくプロダクトの中」。デモ中、写真投入→Visionがライブで推理を返す瞬間を必ず作る。

## 4. ユーザーフロー（最小）
1. 写真を1枚アップロード
2. 「観察を始める」→ AIが観察推理カード（4枚）を生成・表示
3. 各カードを「当たり／外れ」で答え合わせ → 的中率メダル＆称号
（MVPはこの3ステップだけ。プリセット写真・複数枚・共有機能などは時間が余ったら）

## 5. 技術スタック / 制約
- 構成: フロント（HTML/CSS/JS、外部ライブラリなし・Google Fontsのみ）＋ Vercelサーバーレス関数（`/api`）
  - ※APIキーをサーバー側に隠すため、**単一HTMLではなく `/api` を持つ最小構成**にする（ブラウザ直叩きはしない）
- AI: **Claude（Anthropic）をVercelのバックエンドルート経由で呼ぶ。使用AIはAnthropicのみ（Gemini・Groqは使わない）**
  - **呼び出し経路:** フロント → 自前の `/api/claude`（Vercel関数）→ Anthropic API
  - **★Vision必須:** 本アプリは画像入力を使う。`/api/claude` を**画像対応に拡張**する（後述§8）。
    フロントは `{imageBase64, mediaType, userPrompt, systemPrompt}` を送り、サーバ側で
    `messages[].content` に `{type:"image", source:{type:"base64", media_type, data}}` ＋ `{type:"text"}` を組む。
  - **Anthropicエンドポイント（サーバー側＝`/api/claude`内からのみ叩く）:** `https://api.anthropic.com/v1/messages`
  - **必須ヘッダ（`/api/claude`内でAnthropicに付ける。フロントには書かない）:**
    - `x-api-key: process.env.ANTHROPIC_API_KEY`（`Authorization: Bearer` ではない）
    - `anthropic-version: 2023-06-01`
    - `content-type: application/json`
    - ※`anthropic-dangerous-direct-browser-access` は**不要**（ブラウザから直接叩かないため）
  - **モデル文字列:** 当日 docs.claude.com で有効な最新を確認して固定
    - 主力: `claude-sonnet-4-6`（Vision品質と速度のバランス）
    - バックアップ: `claude-haiku-4-5-20251001`（最速・低コスト。混雑時の切替先）
  - **リクエスト要点:** `max_tokens` 必須 / システムプロンプトはトップレベル `system` に渡す
  - **レスポンス取り出し:** `data.content` 配列から `type==="text"` を抽出して連結 → フェンス除去 → `JSON.parse`
  - **APIキー:** **Vercel環境変数 `ANTHROPIC_API_KEY`**（フロントにもリポジトリにも一切出さない）
    - 設定後は再デプロイで反映。従量課金（無料枠なし）。当日までに少額チャージ＋疎通確認。
  - **切替:** `/api/claude` の `const MODEL = "claude-sonnet-4-6"` を1行で Haiku に変えられる構成にする
- 参照skill: `claude-api-skill.md`（Vercel環境変数＋バックエンドルート方式。**画像対応の追記が必要**）

## 9. 既知の課題 / 未決事項
- **答え合わせは自己申告**（写真の持ち主が判定）。客観スコアではない点はデモ上は割り切る。気になるなら「自己採点」と明示。
- **任意写真は精度がばらつく**。情報量の少ない写真だと推理が薄くなる → デモは被写体を決め打ちで安定させる。
- **`max_tokens=1000` で日本語4カードが稀に途中で切れる**可能性 → 本番はトークン調整、またはカードを3枚に。
- Vision対応で `api/claude.js` の改修が前提（テキスト版のままだと画像が通らない）。
- 確信度はAIの自己申告。答え合わせの的中とズレうる（そのズレ自体が面白さでもある）。
